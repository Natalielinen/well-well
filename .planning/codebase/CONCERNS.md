# Технический долг и проблемы

*Последнее обновление: 2026-07-16*

## Технический долг

### Высокий

1. **Монолит в App.tsx (~270 строк)**
   - `App.tsx` содержит всё состояние приложения, бизнес-логику и рендеринг
   - Логика фильтрации, сортировки и проверок дат размещена прямо в компоненте
   - **Риск**: сложность роста, трудности с тестированием
   - **Файл**: `App.tsx`

2. **Логика завершения задачи в Todo.tsx (~120 строк одной функции)**
   - `onTaskComplete` содержит 4 ветви логики с nested Alert-ами
   - `showAlertForExpiredTask` дублирует логику обновления дат
   - **Риск**: баги при изменении логики повторений
   - **Файл**: `components/Todo/Todo.tsx:134-165`

### Средний

3. **Дублирование логики обновления reminder**
   - `getUpdatedTime` в `Todo.tsx:35-41` парсит строку вручную через `split(':')`
   - Аналогичная логика есть в `useNotifications.ts`
   - **Риск**: расхождение при изменении формата времени

4. **Обработка ошибок через console.log**
   - В `storage/todoStorage.ts:10,20` ошибки только логируются
   - В `hooks/useNotifications.ts:105` тоже только `console.error`
   - Пользователь не видит ошибок, данные могут теряться безвозвратно

5. **Отсутствие валидации на уровне типа**
   - `Size` — это `1 | 3 | 5 | 8 | 13`, но нет runtime-проверки
   - `repeatFrequency` хранится как `number`, но нет валидации на отрицательные значения

## Безопасность

| Риск | Уровень | Описание |
|------|---------|----------|
| XSS через задачи | Низкий | Данные из AsyncStorage рендерятся в `Text`, RN экранирует |
| Утечка данных | Средний | AsyncStorage не зашифрован, данные доступны на-rooted устройствах |
| Секреты в коде | Низкий | Ad unit ID хардкожен, но это публичный ID, не секрет |

## Производительность

1. **Полная перезагрузка списка при каждом изменении**
   - `getAllTodos` загружает все задачи из storage и вызывает `setAllTodos`
   - `FlatList` перерендеривается полностью, нет `extraData` оптимизации
   - **Файл**: `App.tsx:115-120`

2. **Лишние re-renders Todo компонентов**
   - `displayedTodos` пересчитывается на каждый рендер через `useMemo`, но зависимость `showAll` триггерит пересчёт
   - **Файл**: `App.tsx:101-113`

3. **Date parsing в render**
   - `isExpired`, `isToday`, `isFuture` парсят даты на каждой итерации `useMemo`
   - **Файл**: `App.tsx:83-96`

## Хрупкие области

1. **Формат дат**
   - Смешение `yyyy-MM-dd`, `dd.MM.yyyy`, ISO строк
   - Ошибка в формате может сломать парсинг и сортировку

2. **Управление уведомлениями**
   - Нет гарантии, что `notificationId` соответствует актуальному запланированному уведомлению
   - При редактировании задачи старый notificationId сбрасывается, но новый может не создаться (если reminder в прошлом)

3. **Асинхронность в useEffect**
   - `useEffect` в `App.tsx:76-81` без cleanup, без обработки ошибок
   - `MobileAds.initialize()` запускается без await

## Известные баги

- В `AddTodo.tsx:108` параметр `event: any` — типовая небезопасность
- В `useNotifications.ts:77` функция объявлена без `export` / `async` — внутренняя, но не очевидная
