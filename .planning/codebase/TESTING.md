## Что тестировать в первую очередь

На основе анализа кода критичные точки для тестирования:

1. **`storage/todoStorage.ts`** — CRUD операции над `TodoItem[]`, edge cases: пустой storage, битый JSON, null значения
2. **`hooks/useNotifications.ts`** — планирование, отмена, обработка разрешений, edge case с прошедшей датой
3. **`App.tsx`** — логика фильтрации (`displayedTodos`), сортировка, навигация по датам
4. **`components/Todo/Todo.tsx`** — сценарии завершения: не повторяющаяся, повторяющаяся не просроченная, просроченная, будущая
