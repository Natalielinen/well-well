# wellWell

## What This Is

React Native / Expo-приложение для управления персональными задачами (todo) с напоминаниями. Пользователь создаёт, редактирует, отмечает выполненные задачи и просматривает их по дням/неделе. Уведомления помогают не пропустить сроки.

## Core Value

Надёжное выполнение задач с своевременными уведомлениями.

## Requirements

See: `.planning/REQUIREMENTS.md`

### Validated

- Просмотр списка задач по дням и неделе
- Создание, редактирование и удаление задач
- Напоминания через push-уведомления
- Фильтрация задач (все / сегодня)
- Повторяющиеся задачи
- Баннерная реклама
- Устранить известные баги из `.planning/codebase/CONCERNS.md` (Validated in Phase 02)

### Active

- [ ] Исправить баг доставки уведомлений (не приходят или с задержкой 2–7 минут)
- [ ] Исправить обновление reminder и notificationId при редактировании повторяющейся задачи
- [ ] Устранить известные баги из `.planning/codebase/CONCERNS.md`
- [ ] Провести рефакторинг монолитного `App.tsx` и тяжёлой логики в `components/Todo/Todo.tsx`
- [ ] Покрыть код unit-тестами

### Out of Scope

- Серверная синхронизация и аккаунты — не требуется для локального todo-трекера
- Реaltime-коллаборация — не входит в текущую ценность продукта

## Context

- Проект существует как локальное приложение без серверной части.
- State и бизнес-логика сейчас сосредоточены в `App.tsx`.
- Есть карта кодабазы в `.planning/codebase/` (архитектура, стек, проблемы).
- Известный технический долг и баги задокументированы в `CONCERNS.md`.

## Constraints

- **Stack**: Expo ~54, React Native 0.81.5, TypeScript strict
- **Хранилище**: `@react-native-async-storage/async-storage`
- **Уведомления**: `expo-notifications`
- **Даты**: `date-fns`
- **Без серверной части**: все данные локальные
